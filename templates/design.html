Diseño del Proyecto

Arquitectura del Sistema

El sistema sigue una arquitectura cliente-servidor con separación clara entre frontend y backend.

Diagrama de Arquitectura

  • Frontend: HTML, CSS, JavaScript
  • Backend: FastAPI (Python)
  • Base de Datos: PostgreSQL
  • Almacenamiento: Supabase

Base de Datos

Modelo de datos relacional con las siguientes entidades principales:

Esquema de Base de Datos

  • Categorías: id, nombre, descripción, activa, media_url, created_at, updated_at, deleted_at
  • Productos: id, nombre, descripción, precio, stock, activo, categoria_id, media_url, created_at, updated_at, deleted_at
  • Clientes: id, nombre, ciudad, canal, media_url, created_at, updated_at, deleted_at
  • Ventas: id, cliente_id, fecha, canal_venta, total, created_at, updated_at
  • Detalles de Venta: id, venta_id, producto_id, cantidad, precio_unitario, subtotal

API REST

Endpoints RESTful para todas las operaciones CRUD:

Endpoints Principales

  • GET /categorias - Listar categorías
  • POST /categorias - Crear categoría
  • PUT /categorias/{id} - Actualizar categoría
  • DELETE /categorias/{id} - Eliminar categoría
  • GET /productos - Listar productos con filtros
  • POST /productos - Crear producto
  • GET /clientes - Listar clientes con filtros
  • POST /ventas - Crear venta
  • GET /ventas - Listar ventas con filtros

Interfaz de Usuario

Diseño responsivo con navegación intuitiva:

Características de la UI

  • Navbar con dropdowns para navegación
  • Formularios para operaciones CRUD
  • Filtros avanzados para listados
  • Diseño responsivo para móviles y desktop
  • Estilos consistentes con CSS
